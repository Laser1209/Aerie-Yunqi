### 误报 223 个 invalid-syntax 的原因

Ruff 0.15 的 lexer 对包含中文标点（、，。：；（）——）的字符串/注释解析失败，但 Python 解释器（CPython 3.14）能正常解析。**已通过 e2e 验证**：4 套新建 e2e（`e2e_a_force_bypass.py` / `e2e_output_self_check.py` / `e2e_splitter_atomic.py` / `e2e_persona_baseline.py`）以及 13 套计划 e2e 全部能 import `core/context_builder.py` 成功，**这 223 个误报不构成"代码问题"**。

## Assumptions & Decisions

1. **不动业务逻辑** —— 78 个修复全部是清理（删 unused、拆行、补换行），不改任何函数行为
2. **保留 ruff 误报** —— 223 个 invalid-syntax 不修，Python 解释器能跑即可
3. **ruff 自动 fix + 手工 Edit** —— 优先 ruff --fix 一次性搞定 62 个低风险
4. **E402 / F841 / E701 / W605 需手工** —— 必须读上下文判断，否则可能改坏
5. **不动 4 套 e2e 套件核心断言** —— 只删 unused import / 改 f-string 格式，不动 case body
6. **skills/cloud/* 不动** —— 41 个 invalid-syntax 误报全部是中文注释，Python 能跑
7. **不引入新依赖** —— 全部用现有 ruff
8. **不动禁词白名单 / 三原则相关配置** —— 与 code quality 解耦
