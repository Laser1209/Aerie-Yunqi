# 数据所有权与迁移纪律
> [!danger] 禁止
> 不删除旧表、旧 uploads 或兼容投影；不跨库 JOIN、双写或复制疑似凭据。

- `aerie.db` 仅 Core 写；`world.db` 仅 Sidecar 写。
- Persona Hub 是写入唯一真源，旧 YAML 仅只读兼容投影。
- 四表与 `chat_log` 兼容周期双写，读路径由 Feature Flag 切换。
- 迁移必须支持 backup、dry-run、checksum、幂等、cursor、断点续跑和守恒校验。
- 回滚优先关闭 Flag 并恢复备份，不删除新表或旧文件。
