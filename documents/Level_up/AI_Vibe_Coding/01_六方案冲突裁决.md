# 六方案冲突裁决
> [!warning] 裁决原则
> 以获批实施计划为唯一执行依据；“不受限制”仅表示上下文与连续输入专项，不绕过安全、工具或权限。

1. 对话升级演进现有 Pipeline 与 Context Builder，不创建平行 v2。
2. 拟人流式复用 SemanticMessageSplitter 与 Persona Pacing。
3. 主动消息复用现有调度器，不新建第二套。
4. 图片资产与副作用归 Core；World 只产生候选。
5. 世界先 InProcess，领域稳定后 Remote Sidecar。
