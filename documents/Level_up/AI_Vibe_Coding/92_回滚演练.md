# 回滚演练
1. 关闭对应 Feature Flag。
2. 恢复数据库备份或切回旧读路径。
3. 验证旧 API、聊天、QQ 与桌面继续可用。
4. 保留新表、元数据、Outbox 和旧文件。
5. 记录恢复时间、数据损失与未完成任务。

## Phase 01：主动消息 P0
- [x] 将 `proactive_delivery_v2` 设为 `false` 时恢复 QQ-only 投递。
- [x] 旧路径不写主动桌面气泡、不发送 `assistant`、`proactive_message` 或 `proactive_delivery` 事件。
- [x] QQ 断线恢复旧 Scheduler 暂停行为；V2 开启时 QQ 断线不阻断本地气泡和系统通知。
- [x] Trigger、Event Engine 生命周期、Desire 属性和 YAML 去重修复不随 Flag 回滚，避免重新引入 P0 断点。
- [x] 自动化证据位于 [[Task 01-baseline]]，未记录消息正文、账号或凭据。

## Phase 02：Actor、Channel 与 Persona 真源
- [x] `identity_contract_v1=false` 时 IdentityResolver 不写 `actor_id`，聊天与长期记忆继续按 legacy `user_id` 寻址。
- [x] 关闭 Flag 后保留 `actors`、`channel_accounts`、可空身份列和新数据，不执行破坏性回滚。
- [x] 旧聊天和长期记忆的规范身份列保持 `NULL`，不猜测历史来源。
- [x] `persona_hub_source_v1=false` 时恢复旧 `persona.yaml` 读写适配路径；Hub JSON 和 active 指针保留。
- [x] Persona JSON 与 `_active.json` 使用原子替换；写盘失败时内存状态不提前切换。
- [x] 自动化证据位于 [[Task 02-baseline]]；Phase 2 当前 Python `328 passed, 6 warnings`，定向 `138 passed, 4 warnings`，Electron `3 passed`。
- [x] 中断恢复后重新验证 Actor Emotion 生命周期、迁移框架兼容、静态 `emotion.get_state(0)` 门禁和完整回归。
- [x] 剩余 `user_id=0` 已分类为全局/系统/legacy 边界；无 Actor 来源的旧 Evolution 记录不做猜测式回填。

## Phase 03：Conversation / Turn / Message / Request
- [x] 自动化验证 `conversation_model_v1=false` 时 Pipeline 不调用规范历史读取或 Turn 镜像入口。
- [x] Flag 关闭后继续使用 legacy `chat_log`，并保持 FULL `LIMIT 20`、BASIC `LIMIT 10` 及原查询参数合同。
- [x] 规范历史读取异常时回退 legacy，旧回复路径继续可用。
- [x] Flag 开启时 FULL/BASIC 均按完整 Turn 读取历史，助手多分段不会从中间截断。
- [x] 新表与规范数据不因关闭 Flag 被删除；仓储禁用态写入为 no-op。
- [x] 规范 Turn 写入中途失败时 SAVEPOINT 整体回滚，不遗留孤立 Conversation、Turn、Request 或 Message。
- [ ] 尚未在真实 Companion 组合根与用户生产数据库执行开关、备份恢复、恢复时间和数据损失演练。
- [ ] `rollback_ready` 保持 `false`，完成真实演练前不得进入 Phase 04。
