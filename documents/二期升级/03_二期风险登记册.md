# Aerie 二期风险登记册

## 1. 文档目的与适用边界

本登记册把二期主计划中的 `R-01` 至 `R-17` 转换为可执行的责任、触发、控制、应急、回滚和证据合同，并完整关联 `P2-00-B02` 发现的八个阻断。它只依据独立证据更新单项阻断状态，不关闭 G0、不批准进入 `P2-01`，也不授权修改产品配置、依赖、数据库、Git 历史或远端状态。

本文件的事实来源按以下顺序解释：

1. `Aerie_项目二期升级更新计划.md` 的强制原则、风险评分、测试、回滚和门禁。
2. `07_二期任务编排.md` 的角色、批次、阶段门和证据代码。
3. `00_二期基线与环境冻结.md` 的冻结基线、工作区归属和 Git 边界。
4. `04_二期测试与证据索引.md` 的 B02 实测终态和阻断台账。
5. 2026-07-21 对当前共享工作区和三个 Git 单元所做的只读复核。

安全约束：本登记册不保存、不回显任何候选密钥原文，只使用风险描述、路径级事实、状态和阻断编号。供应商后台的撤销或轮换状态不能仅凭本机代码独立证明。

## 2. 评分、状态与复核规则

概率 `P` 与影响 `I` 均为 1-5，分数为 `P x I`：15-25 为高，8-14 为中，1-7 为低。

| 状态 | 含义 | 退出条件 |
|---|---|---|
| `Open` | 风险存在，尚未达到触发阈值，或控制/证据尚不完整 | 控制与证据完成后转 `Mitigating` 或 `Closed` |
| `Triggered` | 当前事实已经达到一个或多个触发阈值 | 执行应急动作并进入 `Mitigating` |
| `Mitigating` | 修复、复测或回滚正在进行 | 关闭条件全部满足，或形成有期限例外 |
| `Accepted` | PO 对限定范围、期限和规避措施作出书面例外 | 到期复审；不得自动续期 |
| `Closed` | 关闭条件及独立 Review 均有可追踪证据 | 后续回归失败时立即重新打开 |

执行规则：

- `Owner` 对预防、监控和处置负责；`Reviewer` 必须独立确认结论，不能由 Owner 自批。
- 任一 `Blocker` 或 `Critical` 风险不得以口头接受关闭；秘密、权限绕过和包夹带秘密不允许普通产品例外。
- `last_reviewed_at` 是本轮事实复核日期；`next_review_at` 是最晚复核点。阻断、阈值、代码、配置、依赖或工作区归属变化时应提前复核。
- 没有 Evidence 的“未发现问题”只能记为 `Open`，不能记为 `Closed`。
- `DEV-P2-00-001` 只接受历史 mixed commit 的事实，不豁免未来提交原子性，也不改变 `R-13` 的状态。
